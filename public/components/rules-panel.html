NFS-e Nacional

Regras operacionais para distribuição de XMLs

Fontes consultadas Portal Gov.br NFS-e - Documentação Atual Manual de API ADN para Contribuintes Manual de API ADN para Municípios
01

Consulta por NSU

A distribuição é feita a partir de um Número Sequencial Unico. O sistema deve guardar o último NSU recebido e continuar do próximo ponto, sem recomeçar toda a varredura sem necessidade.

02

Lote máximo

A documentação do ADN descreve retorno em lotes de até 50 documentos fiscais por chamada. A tela usa paginação menor para visualização, mas a varredura respeita a dinâmica de lotes da API.

03

Ordem dos documentos

O NSU segue a disponibilização no ambiente nacional. Ele não deve ser tratado como ordenação perfeita por data de emissao; eventos e notas podem aparecer conforme recepção pelo ADN.

04

Atores autorizados

As APIs de contribuinte permitem consulta de documentos em que o CNPJ figure como emitente/prestador, tomador ou intermediario, conforme autorizacao do certificado usado na conexao.

05

NFS-e e eventos

Eventos vinculados, como cancelamento, fazem parte do ciclo de vida da NFS-e e podem aparecer na distribuição. A lista consolida a nota principal quando há documento e evento relacionados.

06

Certificado digital

A chamada usa mTLS com certificado ICP-Brasil A1 em arquivo PFX/P12. O certificado define a identidade fiscal e limita quais documentos podem ser acessados pela conexão.

07

Consumo indevido

Consultas repetidas, muito rápidas ou insistentes após fim de fila podem gerar rejeição/bloqueio temporário. Use a pausa por lote e evite reiniciar varreduras completas sem necessidade.

08

Persistência local

A visualização de XMLs é feita a partir do Supabase, com limit/offset. Isso permite navegar milhares de registros por unidade sem carregar todo o histórico no navegador.

Fluxo recomendado de operação

1. Cadastre o certificadoValide senha, CNPJ e validade antes de iniciar.
2. Selecione a unidadeUse o CNPJ da unidade desejada para filtrar a base persistida.
3. Use último recebidoRetome do NSU salvo para evitar chamadas redundantes.
4. Execute com pausa fixaO sistema aguarda 2 segundos entre lotes para reduzir pressão na API.
5. Navegue pela listaA tabela carrega 10 XMLs por páginas diretamente do Supabase.
