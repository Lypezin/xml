Regras operacionais ADN

Boas práticas para distribuição de NFS-e, consumo da API e operação segura no dia a dia.

Fontes oficiais
  • Portal Gov.br NFS-e — documentação atual
  • Manual de API ADN para Contribuintes

Princípios da distribuição
01 Core

Consulta por NSU

A distribuição usa Número Sequencial Único. Guarde o último NSU recebido e continue do próximo ponto — não recomece a varredura sem necessidade.

02

Lote máximo

Até 50 documentos por chamada na ADN. A UI pagina 10 itens; a varredura respeita os lotes da API.

03

Ordem dos documentos

NSU segue a disponibilização nacional — não é ordenação perfeita por data de emissão.

04

Atores autorizados

Prestador, tomador ou intermediário conforme o certificado mTLS da conexão.

05

NFS-e e eventos

Cancelamentos e eventos fazem parte do ciclo. A lista consolida a nota principal quando há vínculo.

06 Segurança

Certificado digital

mTLS com A1 (PFX/P12) ICP-Brasil. O certificado define identidade e o recorte acessível.

07 Risco

Consumo indevido

Consultas rápidas demais ou após fim de fila podem bloquear o CNPJ. Use pausa de 2s entre lotes.

08

Persistência

A tabela lê o banco com limit/offset — milhares de XMLs sem carregar tudo no browser.

Fluxo recomendado
  1. 1
    Cadastre o certificado Valide senha, CNPJ e validade antes de iniciar.
  2. 2
    Selecione a unidade Filtre a base pelo CNPJ da unidade desejada.
  3. 3
    Use o último NSU Retome do NSU salvo para evitar chamadas redundantes.
  4. 4
    Execute com pausa 2 segundos entre lotes reduz pressão na API nacional.
  5. 5
    Navegue a lista 10 XMLs por página, direto do banco de dados.
